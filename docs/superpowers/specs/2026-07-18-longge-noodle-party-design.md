# 龙哥炸酱面聚会邀请函：产品与视觉设计

## 1. 项目目标

制作一个适合微信等聊天工具分享、以手机浏览为主的单页互动邀请函。页面邀请朋友于 **2026 年 7 月 19 日（周日）** 到家中吃炸酱面，并通过手账式折叠流程展示聚会地点、到场时间、人物阵容和六个活动环节。

成品为纯静态网页，可直接部署到 GitHub Pages；无需登录、后端或数据库。

## 2. 已确认信息

- 日期：2026 年 7 月 19 日（周日）
- 普通到场时间：15:00–16:00
- `AI Native 连续创业者` 到场时间：17:00 特别登场
- 地址：北京市昌平区风雅园一区 15 号楼 1 单元 303
- 导航：点击后使用 Google Maps 搜索上述完整地址
- 隐私规则：页面、代码、图片替代文本和分享文案中均不得出现图 3 人物的真实姓名，只能使用 `AI Native 连续创业者`
- 人物关系：图 1–4 是客人，图 5–6 是主人

## 3. 产品方向

采用「聚会手账折叠册」方案，并加入轻量任务印章：

- 手机纵向长页，像朋友亲手制作的周日聚会手账。
- 六个活动环节用可折叠卡片呈现。
- 同时只展开一个环节；新环节展开后进入合适视野。
- 已浏览或完成互动的环节获得手账印章。
- 六个环节全部浏览后解锁底部六人合影。
- 互动是惊喜层，不阻碍用户直接查看日期、时间、地址和流程。

## 4. 视觉系统

### 4.1 核心气质

温暖、松弛、可爱、有私人感；像朋友为这次聚会专门制作的手账，而不是商业活动页或 SaaS 落地页。

### 4.2 色彩与材质

- 纸张底色：奶油纸 `#FFF8E8`
- 正文与描边：炸酱棕 `#663C2A`
- 主强调：番茄红 `#E85D4A`
- 次强调：蛋黄黄 `#F5C84C`
- 辅助色：黄瓜绿 `#83A95C`、便签粉 `#F5B8C1`
- 背景带低对比纸张颗粒；卡片使用撕纸边、和纸胶带、印章、虚线箭头、票根和少量食物涂鸦。
- 每个可视区域最多使用两类装饰元素，正文优先。

### 4.3 字体与排版

- 正文使用系统圆体优先字体栈，保证加载速度和中文可读性。
- 标题可使用项目内置的开源手写风字体；若字体加载失败，退回圆体。
- 手机端为主要断点；桌面端保持约 430px 的手账主体居中，并允许装饰轻微超出纸张边缘。
- 不使用紫色渐变、玻璃拟态、3D 塑料质感或标准化 SaaS 大圆角卡片。

### 4.4 总体视觉提示词

> 温暖的家庭聚会手账，奶油色再生纸背景，蜡笔与彩铅质感，撕纸边缘、和纸胶带、便签、印章、虚线箭头和食物涂鸦；不规则但清晰的拼贴排版，番茄红和蛋黄黄点缀，带有真实朋友制作的私人感。可爱但不幼稚，热闹但不杂乱，适合手机纵向浏览。

## 5. 人物重绘规范

六张来源头像不直接作为页面贴图，而是重绘为同一套透明背景手账角色贴纸。应保留每张原图的核心识别点，并统一角色比例、描边、材质、阴影方向和色彩明度。

### 5.1 通用生成提示词

> 六位朋友的可爱手账角色贴纸，日系蜡笔和彩铅质感，奶油纸色调，深棕色不规则描边，圆润简化造型，轻微纸张颗粒，番茄红与蛋黄黄点缀，温暖家庭聚会氛围。角色比例、光影、线条粗细完全统一，保留参考头像的核心辨识特征。可爱但不幼稚，不生成文字，透明背景，完整角色，无裁切。

### 5.2 角色定义

1. **黄色的猫 / 炸酱面主理人 / 客人**：保留黄色花朵颈圈和大眼睛，增加围裙与一碗炸酱面。
2. **可爱的粉色小熊 / AI Builder、产品负责人 / 客人**：保留粉白圆润造型与粉色衣服，抱电脑和产品便签。
3. **三个小幽灵 / AI Native 连续创业者 / 客人**：必须保持三个为一组，在纸杯或容器里活泼乱飞；禁止出现真实姓名。
4. **白色毛绒角色 / AI 运营负责人 / 客人**：保留头部装饰的鲜艳节奏感，手拿扩音器与活动清单。
5. **戴圆眼镜的小狗 / 小红书千粉博主 / 主人**：保留圆眼镜、棕色毛衣和电脑，增加小型相机。
6. **夕阳背影人物 / 家庭煮夫 / 主人**：保留背影、夕阳暖光与安静气质，增加围裙和锅铲。

### 5.3 负向规则

> 不要 3D 塑料质感，不要写实摄影，不要厚重阴影，不要复杂背景，不要文字，不要水印，不要擅自增加或减少角色，不要改变角色物种和关键配饰，不要裁切头部或手持物。

如一次生成无法保持六位角色一致，使用同一风格参考图逐个生成，最后通过统一白边、纸张阴影和色彩校正收敛风格。

## 6. 页面结构与文案

### 6.1 顶部工具

- 静音/声音按钮：默认静音，用户主动开启后才播放轻量反馈音。
- 分享按钮：优先调用 Web Share API；不支持时复制当前链接并显示成功提示。

### 6.2 封面与门铃

- 主标题：`周日来家里吃面吧！`
- 副标题：`龙哥炸酱面 · 好友聚会`
- 日期：`2026.07.19 周日`
- 两位主人位于门后；点击门铃后探头迎客，面条蒸汽组成短暂的欢迎图案。
- 门铃只是彩蛋，不阻塞继续阅读。

### 6.3 到场阵容

- 四位客人位于 `TODAY'S GUEST LIST` 区域。
- 两位主人位于 `今日主人组 · 已就位 ✓` 区域。
- 客人角色贴纸依次落下；点击角色显示完整称号。
- `AI Native 连续创业者` 可先显示为三只幽灵剪影，点击后显示 `17:00 特别登场`。

### 6.4 时间便签

- `大部队：15:00–16:00`
- `AI Native 连续创业者：17:00 特别登场`

### 6.5 地址卡

- 显示完整地址：`北京市昌平区风雅园一区 15 号楼 1 单元 303`
- 主按钮：`Google 地图导航`
- 次按钮：`复制地址`
- Google Maps 使用带 URL 编码地址的搜索链接，并以新标签页打开。
- 导航或剪贴板 API 失败时，地址文本仍可长按选择，并出现清晰提示。

### 6.6 TODAY'S PLAN

六张折叠卡依次排列，第一张默认展开。标题区始终显示 `PART 编号 + 名称 + 一句话预告 + 展开状态`。

#### PART 01 · 到家集合

- 文案：`放下包包，先随便坐！`
- 展示零食袋、糖果罐、饮料和水果篮。
- 点击零食袋后，少量零食贴纸弹出并自然落下。
- 正文：`食材和零食都准备好啦，大家带着嘴来就好！`
- 完成印章：`顺利会师`

#### PART 02 · 龙哥开饭

- 文案：`今日主角：龙哥牌炸酱面`
- 用户依次点击黄瓜丝、胡萝卜丝、豆芽和炸酱，将配菜加入面碗。
- 配齐后面碗冒出爱心热气，并自动盖章。
- 连续点击面碗三次，黄色猫偷偷多加一勺炸酱。
- 完成印章：`香迷糊了`

#### PART 03 · 咔嚓留念

- 文案：`吃面不拍照，等于没吃到！`
- 点击快门后短暂闪光，六人合照从相机下方缓慢吐出。
- 提供 `保存合影` 与 `再拍一张`；保存失败时提示用户长按图片保存。
- 完成印章：`今日份回忆`

#### PART 04 · 水果时间

- 文案：`饭后水果，给胃留个甜甜的结尾。`
- 点击西瓜后切成六瓣，六位角色依次拿走一瓣。
- 完成印章：`水果自由`

#### PART 05 · 麦克风时间

- 标题补充：`吐槽大会 × 家庭脱口秀`
- 点击麦克风，随机选择一位角色接麦，并显示一条气泡：
  - `这个可以说吗？`
  - `展开讲讲！`
  - `今天不许端水！`
  - `掌声在哪里？`
  - `刚才那段掐了别播！`
- 连续点击可轮换角色和台词，避免立即重复上一条。
- 完成印章：`全场最佳`

#### PART 06 · 快乐散场

- 文案：`吃饱了，也笑累了。`
- 用户横向拖动月亮或点击终点，将页面局部场景从黄昏切换为夜晚。
- 四位客人挥手离开，两位主人站在门口送别。
- 点击门口小灯显示：`下次还来吃面！`
- 完成印章：`平安到家`

### 6.7 隐藏合影与结尾

- 六个环节均被展开过后，底部隐藏夹层自动解锁。
- 展示六位统一画风角色的合影。
- 文案：`普通的一顿饭，因为是我们，就变成了值得保存的一天。`
- 收尾：`明天见，不见不散！`

## 7. 交互状态规则

- 折叠面板同时只展开一个；支持键盘与触摸操作。
- `已翻阅`：面板被展开过；`已完成`：该面板核心互动被触发。两种状态视觉区分。
- 隐藏合影的解锁条件是六个面板都已翻阅，不强制完成小游戏。
- 状态保存在当前浏览器的 `localStorage`；再次打开可保留印章和解锁结果，并提供 `重新玩一遍`。
- 互动按钮都具备可读标签、明确焦点态和至少 44px 的触摸区域。
- 动画尊重 `prefers-reduced-motion`；减少动态时直接切换最终状态。
- 不自动播放声音。声音仅作为点击、印章、快门等操作的短反馈，关闭后立即静音。
- 页面不收集个人信息，不接入统计、Cookie 或第三方追踪。

## 8. 技术设计

### 8.1 实现方式

使用原生 HTML、CSS 与 JavaScript，避免构建依赖；所有资源使用相对路径，可直接作为 GitHub Pages 静态站点发布。

建议文件边界：

- `index.html`：语义结构、固定文案、SEO 与分享元信息
- `styles.css`：设计令牌、布局、纸张纹理、人物贴纸、折叠与动画
- `app.js`：折叠控制、小游戏状态、分享、复制、声音与本地存储
- `assets/characters/`：六位角色与合影 WebP/PNG
- `assets/icons/`：必要的自绘 SVG 小图标
- `assets/audio/`：可选且压缩后的短反馈音

### 8.2 性能与兼容

- 首屏人物图与封面优先加载，流程内图片懒加载。
- 图片优先使用 WebP，并保留透明通道；控制首屏资源总量。
- 支持现代 iOS Safari、Android Chrome 和桌面主流浏览器。
- 无 JavaScript 时仍显示完整日期、地址、人物和全部流程文案。

### 8.3 分享与预览

- 设置中文页面标题、描述、主题色和 Open Graph 元信息。
- 制作 1200×630 的分享预览图，避免在聊天软件中只显示空白链接。
- GitHub Pages 基础路径可能包含仓库名，所有资源禁止使用以 `/` 开头的绝对站点路径。

## 9. 验收标准

1. 在 375px 宽手机上无需横向滚动，地址和主按钮清晰可见。
2. Google 地图按钮携带完整、正确编码的地址。
3. 页面任何位置、代码文案、替代文本与元信息都不出现被禁止的真实姓名。
4. 六位人物身份和客主关系正确，且图 3 保持三个小幽灵为一组。
5. 六个折叠面板可操作，同时仅展开一个；六个均翻阅后解锁合影。
6. 六个环节的核心互动可在触屏和鼠标上完成，并有可跳过的最终状态。
7. 分享、复制地址和保存合影在 API 不可用时均有降级提示。
8. 关闭声音后无任何音频播放；首次访问不自动播放。
9. 开启减少动态效果后，页面仍完整可用且不会运行大幅动画。
10. 静态文件可通过 GitHub Pages 从仓库子路径正常加载。

## 10. 明确不做

- 不制作报名、人数统计、评论或聊天功能。
- 不接入后端、数据库、地图 SDK 或定位权限。
- 不根据实时位置规划路线，只打开 Google Maps 地址搜索。
- 不展示受保护的真实姓名。
- 不为装饰而加入持续循环、影响阅读的高频动画。
